# Distill Skill — Design Document

## Overview

The `/distill` skill is a Claude Code custom skill for the vibe-flow project. It captures and organizes information from a Claude Code conversation session into a structured Markdown document, filtered by a user-specified topic.

## Problem

When using Claude Code for Q&A learning, concept exploration, or software design discussions, valuable information stays trapped in the session chat history. There is no easy way to extract and preserve that knowledge as a reusable document.

## Solution

A project-scoped skill that, when invoked with a topic argument, scans the entire conversation, extracts relevant information, and writes a well-organized Markdown document.

## Design Decisions

### Project-scoped, not personal
The skill lives in `.claude/skills/distill/SKILL.md` (inside the repo), not in `~/.claude/skills/`. This keeps it tied to the project and available to anyone working on the repo.

### Manual invocation only
`disable-model-invocation: true` is set in the frontmatter. Claude will never auto-trigger this skill — it only runs when the user explicitly types `/distill <topic>`. This prevents unintended document generation during normal conversation.

### Topic as a filter argument
The skill accepts `$ARGUMENTS` as the topic string. Since topics can contain spaces (e.g., `/distill auth design for API`), the entire argument string is captured as one value using `$ARGUMENTS` rather than positional args like `$0`, `$1`.

### Logical ordering over chat ordering
The skill instructs Claude to reorganize extracted content in logical order — grouping related ideas and building a coherent structure — rather than preserving the chronological order of the conversation. This produces cleaner, more readable documents.

### Adaptive document format
Rather than prescribing a fixed template, the skill lets Claude choose the best format for the content. Possible formats include:

- Design doc
- Concept notes
- Decision log
- Tutorial/guide
- Comparison/analysis

This flexibility ensures each document is structured in the way that best serves its content.

### Output location and naming
- **Directory**: `docs/distilled/` — a dedicated folder that clearly separates AI-generated documents from hand-written docs.
- **Filename**: Auto-generated by Claude in kebab-case, derived from the topic (e.g., `auth-design.md`).
- **Conflict handling**: If the filename already exists, a datetime suffix (`-YYYYMMDD-HHMMSS`) is appended to the new file.

## Skill File Structure

```
.claude/skills/distill/SKILL.md
```

### Frontmatter

| Field | Value | Purpose |
|-------|-------|---------|
| `name` | `distill` | Skill identifier |
| `description` | Distill conversation into a structured document by topic | Human-readable description |
| `disable-model-invocation` | `true` | Prevent auto-triggering |

## Usage

```
/distill <topic>
```

**Examples:**

```
/distill auth design for API
/distill python async concepts
/distill database migration strategy
```

## Output

A single Markdown file written to `docs/distilled/<auto-generated-name>.md`, with a summary of the file path and captured content reported back to the user.
